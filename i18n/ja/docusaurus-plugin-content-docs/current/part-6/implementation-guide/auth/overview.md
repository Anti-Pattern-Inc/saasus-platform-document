SaaSus Platform 認証APIを使って、独自のログイン画面からサインイン処理を実装するための概要を説明します。

:::info
サンプルコードの具体的な実装内容については、[サインイン処理の実装](/docs/part-6/implementation-guide/auth/sign-in)をご参照ください。
:::

## 認証APIとは

認証APIは、SaaSus Platform が提供する標準のホステッドログイン画面を使用せず、SaaS プロバイダーが独自に構築した UI からログイン機能を実装するための API です。

サーバーサイドから SaaSus Auth API（`/sign-in` および `/sign-in/challenge`）を呼び出すことで、ID トークン・アクセストークン・リフレッシュトークンを取得でき、以降の API 呼び出しや認可に利用します。

本ドキュメントでは、この認証APIを使って「独自ログイン画面 → サーバーサイド → SaaSus Auth API」の流れでサインイン処理を実装する方法を説明します。

## 認証APIでできること

### 独自ログイン画面からのメールアドレス＋パスワード認証

SaaSus 標準ログイン画面を利用せず、自社ブランドの UI からログイン処理を行えます。ログイン画面のデザインやUXをカスタマイズできます。

### SaaSus に登録済みユーザーのサインイン

SaaSus 運用コンソールや自社管理画面で登録したユーザーを、API 経由でログインさせられます。

### トークン（ID／アクセス／リフレッシュトークン）の取得

正常に認証が完了すると、SaaSus API 呼び出しやアプリ内認可に利用できる各種トークンが返却されます。

### 標準ログイン画面では対応しづらいユースケースへの対応

ダミーのメールアドレス形式 ID を使ったログインなど、SaaS 側の要件に合わせた柔軟なログイン方式を実装できます。

## 認証フローとアーキテクチャ

認証APIは、Secure Remote Password（SRP）プロトコルに基づいた **2段階認証フロー** です。単にパスワードを送信するのではなく、チャレンジ＆レスポンス方式で安全に認証を行います。

### 認証フロー概要

```
Frontend (React)            Backend (Go)               SaaSus Auth API
     |                           |                           |
     | 1. email + password       |                           |
     |-------------------------->|                           |
     |                           | 2. POST /sign-in          |
     |                           |  (SRP_A を計算して送信)     |
     |                           |-------------------------->|
     |                           |                           |
     |                           | 3. チャレンジレスポンス      |
     |                           |  (PASSWORD_VERIFIER 等)    |
     |                           |<--------------------------|
     | 4. チャレンジパラメータ     |                           |
     |<--------------------------|                           |
     |                           |                           |
     | 5. チャレンジレスポンス送信  |                           |
     |-------------------------->|                           |
     |                           | 6. POST /sign-in/challenge |
     |                           |-------------------------->|
     |                           |                           |
     |                           | 7. トークン返却            |
     |                           |<--------------------------|
     | 8. トークン (Cookie)      |                           |
     |<--------------------------|                           |
```

### フローの詳細

1. **認証開始**: フロントエンドからバックエンドにメールアドレス（または ID）とパスワードを送信します。
2. **SRP 計算**: バックエンドが SRP_A を計算し、SaaSus Auth API の `/sign-in` を呼び出します。
3. **チャレンジ受信**: SaaSus Auth API からチャレンジ（`PASSWORD_VERIFIER` など）が返されます。
4. **チャレンジ転送**: バックエンドがチャレンジパラメータをフロントエンドに返却します。
5. **チャレンジ応答**: フロントエンドがチャレンジレスポンス（`PASSWORD_CLAIM_SIGNATURE` など）をバックエンドに送信します。
6. **チャレンジ検証**: バックエンドが SaaSus Auth API の `/sign-in/challenge` を呼び出します。
7. **トークン発行**: 認証成功時、ID トークン・アクセストークン・リフレッシュトークンが返却されます。
8. **トークン保存**: バックエンドが HttpOnly Cookie でトークンをフロントエンドに返します。

:::info SRP プロトコルのメリット
SRP（Secure Remote Password）プロトコルでは、パスワードそのものをネットワーク上に平文で送信しません。ハッシュ化や SRP 計算を通じて認証を行うため、中間者攻撃に対してもセキュアな認証が実現できます。フロントエンドだけで SRP 計算を完結させるのは複雑なため、バックエンドが SaaSus API との通信を仲介する構成が推奨されます。
:::

### チャレンジの種類と分岐処理

SaaSus Auth API から返されるチャレンジには、以下の種類があります：

| チャレンジ名 | 説明 | 処理 |
|---|---|---|
| `PASSWORD_VERIFIER` | 通常のパスワード検証 | チャレンジレスポンスを送信してトークンを取得 |
| `NEW_PASSWORD_REQUIRED` | 初回ログイン時のパスワード変更 | 新しいパスワードの入力を求め、再度チャレンジを送信 |

## ID とメールアドレスの併用ログイン

SaaSus Platform はユーザー識別子として **メールアドレス形式** を必要としますが、SaaS の要件として「社員ID」や「ユーザーID」（例: `employee001`）でのログインを提供したいケースがあります。

本サンプルアプリケーションでは、バックエンドで以下の変換ロジックを実装することで、**「ID またはメールアドレス」のハイブリッド認証** を実現しています。

### 変換ロジックの概要

1. ユーザーが入力したログイン ID 文字列を解析
2. **メールアドレス形式**（`@` を含む）の場合 → そのまま SaaSus API へ送信
3. **ID 形式**（`@` を含まない）の場合 → システムで定めたダミードメイン（例: `@example.auth`）を付与してメールアドレス形式に変換し、SaaSus API へ送信

これにより、エンドユーザーは裏側のメールアドレス形式を意識することなく、ID だけでログインが可能になります。

:::tip 実装のポイント
ID 形式でのログインを利用する場合、SaaSus Platform にユーザーを登録する際にも同じダミードメインを付与したメールアドレス形式で登録する必要があります。登録時と認証時でドメインが一致しないと、ユーザーが見つからずログインに失敗します。
:::

## サンプルアプリケーションの構成

本実装ガイドのサンプルアプリケーションは、以下の構成で実装されています：

| 項目 | 技術スタック |
|---|---|
| フロントエンド | React + TypeScript |
| バックエンド | Go |
| トークン管理 | HttpOnly Cookie |
| セキュリティ | CSRF 対策 |

### 実装されている画面・機能

- **ログイン画面**: メールアドレス（または ID）＋パスワードの入力フォーム
- **新パスワード設定画面**: 初回ログイン時のパスワード変更
- **サインインAPI**: SaaSus Auth API との SRP 認証フロー
- **トークン管理**: HttpOnly Cookie による安全なトークン保存
- **ログアウト処理**: Cookie クリアによるセッション終了

具体的な実装内容は、[サインイン処理の実装](/docs/part-6/implementation-guide/auth/sign-in)をご覧ください。
